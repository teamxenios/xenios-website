# Research Guides

Member-only evidence Guides. The full library is visible to active members only. The shared
Research password unlocks the entrance and application layer, never this tree.

## Layout

```text
individual/<compound-slug>/    20 individual compound Guide packets
blends/<blend-slug>/            6 blend Guide packets
member-faq/                     member education FAQ library
```

Every Guide packet contains nine files:

```text
SOURCE_PLAN.md                 search strategy and what remains to be searched
SOURCE_REGISTRY.md             every source, with verification status
CLAIM_TABLE.md                 claim-level evidence grades and prohibited claims
CONTRADICTIONS.md              where sources disagree
GUIDE_DRAFT.md                 the member-facing Guide
FAQ_DRAFT.md                   member questions answered honestly
QUALITY_AND_DOCUMENTATION.md   what a supplier must satisfy, and the current unmet state
REGULATORY_STATUS.md           dated, jurisdiction-specific, sourced
EDITORIAL_REVIEW_CHECKLIST.md  the sign-off gate, every box starting unchecked
```

## Status of everything in this tree

Every file here is an **AI draft at workflow state `draft`**. Nothing in this tree is approved,
published, or member-visible. Nothing here has passed scientific, claims, quality, legal, or
founder review.

Before any file publishes, a human must verify every citation independently. Citations marked
`UNVERIFIED` have not been retrieved from a primary source and must not be relied on.

## Rules that bind every file here

1. No dosing, amounts, concentrations, frequency, or administration information.
2. No guaranteed outcomes, and no blanket safety claims.
3. Every preclinical finding names its species or model in the same sentence.
4. Every regulatory statement carries a date, a jurisdiction, and a source URL.
5. Evidence for individual ingredients does not establish evidence for a blend.
6. AI drafts never publish automatically. Samuel Boadu is founder reviewer and publisher.

See `docs/research-content/GUIDE_CONTENT_SCHEMA.md` for the full contract and
`docs/research-content/EDITORIAL_WORKFLOW.md` for the lifecycle gates.
