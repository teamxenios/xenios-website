# Final Versioning and Publication Standard

- Final package version: **1.0**.
- Effective date: **July 22, 2026**.
- A published document is immutable. Any text change creates a new semantic version and content hash.
- The signing record stores member ID, document ID, version, content hash, typed legal name, acceptance event, timestamp, IP address, user agent, and electronic-consent version.
- Only published and effective versions may be signed.
- Material revisions require affirmative reacceptance when required by the document, policy, or applicable law.
- Superseded documents remain available in the member's signed-document archive.
