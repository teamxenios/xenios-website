## 1. Purpose

This matrix is the internal map of the regulated areas the Xenios Research Founding Membership touches, the federal authority that governs each area, the state-law dimension counsel must review, and the library documents that implement the related obligations.

This matrix is a framework, not a set of conclusions. It identifies where legal analysis is required. It does not perform that analysis. Every row carries the status "Pending counsel" until qualified counsel completes the analysis for that row and records the result here under the Document Control SOP (XR-REG-008).

## 2. How to read this matrix

1. "Federal authority" names the agency and the rule or statute by name. It does not cite section numbers. Counsel supplies exact citations during review.
2. "State-law dimension" names statute families, not individual statutes. Where a specific state is named, it is an example flagged for counsel, not a determination that the state's law applies or does not apply.
3. "Implementing library documents" lists the document keys in this library that are designed to carry the related obligations. Listing a document does not mean the obligation is satisfied. It means that document is where counsel's conclusions must land.
4. "Analysis status" is a conclusion cell. In this draft, every conclusion cell says pending. No cell in this document states that a law applies, does not apply, or is complied with.
5. The 21+ policy, the private membership structure, and the application gate may narrow some obligations. None of them is assumed to eliminate any obligation anywhere in this matrix.

## 4. Standing notes on the matrix

1. No row above concludes that any law applies or does not apply. "Pending counsel" is the only status this draft is permitted to carry.
2. Where two rows overlap (for example, consumer health data and biometric data both touch the tracker), counsel should resolve the overlap in the row-specific analysis and record which document carries which obligation.
3. Earlier drafts exist in this worktree under `docs/compliance/`, `docs/privacy/`, `docs/security/`, and `docs/risk/` (for example HIPAA_APPLICABILITY_ANALYSIS.md and FTC_HBNR_APPLICABILITY_ANALYSIS.md). This matrix does not supersede them; counsel should reconcile them with XR-POL-008 and XR-POL-009 and designate the surviving documents.
4. The Quantum lane is Coming Soon and non-commerce. Its classification questions live in XR-QTM-003 through XR-QTM-008 and are intentionally not resolved here.
5. Any change to a program state (a new SKU category, a new channel, commerce activation, a new partner type) reopens the affected rows. The Document Control SOP (XR-REG-008) governs how a reopened row is versioned.

### 5.3 Per-state questions counsel must answer

1. May Xenios lawfully ship each active SKU category (supplements; research products) to consumers in this state, and under what conditions?
2. Does this state's automatic renewal law impose disclosure, consent, reminder, or cancellation mechanics beyond the current federal-baseline design in XR-MEM-003 and XR-COM-002?
3. Does this state's privacy or consumer health data law grant rights or require consents beyond the current design in XR-PUB-004, XR-TRK-001, and XR-MEM-012, and does it require processor contracts with any current vendor?
4. Does identity verification or any tracker media feature trigger this state's biometric statute, and if so, what consent, retention, and destruction terms are required in XR-MEM-011 and XR-TRK-007?
5. Does any plan, Guide, coaching, or recommendation feature trigger professional licensure (medical, dietetics, nutrition) in this state as currently designed?
6. Does this state restrict or condition the partner compensation model in XR-AFF-007, or require business opportunity or anti-pyramid registrations or disclosures?
7. What retention periods does this state require for the record families in XR-POL-005, and do any exceed the current placeholders?
8. Does this state require registration, licensing, or assumed-name filings for the entity, [ENTITY], to operate this program as designed?
